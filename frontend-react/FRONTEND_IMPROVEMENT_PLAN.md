## 1. الأزرار/العناصر التي لا تعمل

### أ) النصوص المقترحة (Suggested Prompts) عند فتح الدردشة

**المشكلة:** البطاقات في `MessageList.js` (مثل "ما هي خدمات وريد الصحية؟"، "كم سعر تحليل فيتامين د؟") هي عناصر `<div>` بدون `onClick` — لا تفعل شيئاً عند النقر.

**الموقع:** `src/components/MessageList.js` أسطر 88–99

**الحل:**
- إضافة prop `onSuggestedPromptClick` لـ `MessageList`
- تمريرها من `ChatWindow` ← `App.js` (من `handleSendMessage`)
- تحويل كل `prompt-card` إلى `<button>` أو إضافة `onClick` ينفّذ إرسال النص

```jsx
// MessageList - إضافة onClick للبطاقات
<button
  type="button"
  className="prompt-card"
  onClick={() => onSuggestedPromptClick('ما هي خدمات وريد الصحية؟')}
>
  {formatArabicText('ما هي خدمات وريد الصحية؟')}
</button>
```

**المشكلة:** الزر معطّل (`disabled`) وعنوانه "Coming soon".

**الموقع:** `src/components/ChatInput.js` أسطر 71–78

**الحل (خياران):**

   - إضافة endpoint جديد في الباك اند لقبول رسائل تحتوي صوراً (مثلاً base64 أو رفع ملف)

   - عند اختيار صورة: إما إرفاقها مع النص أو إرسالها كرسالة منفصلة حسب تصميم الـ API

**ملاحظة:** الـ API الحالي (`/api/conversations/{id}/messages`) يقبل `content` نصياً فقط. دعم الصور يحتاج تعديل الباك اند.

### د) التسجيل (Register)

**المشكلة المحتملة:** قد يفشل التسجيل لأحد الأسباب التالية:

1. **قاعدة البيانات غير مفعّلة:** إذا لم يكن `DATABASE_URL` مضبوطاً في `.env`، يرجع الباك اند 503: "قاعدة البيانات غير مفعّلة. التسجيل والدخول غير متاحين."
2. **البريد مسجّل مسبقاً:** يرجع 400: "البريد الإلكتروني مسجّل مسبقاً."
3. **مشكلة الـ Proxy:** في التطوير، الـ proxy يوجّه الطلبات إلى `http://127.0.0.1:8000`. تأكد أن الباك اند يعمل على المنفذ 8000.

**الموقع:** `src/components/Register.js` — يستخدم `api.post("/api/auth/register", {email, password})`

**الحل:**
- التأكد من ضبط `DATABASE_URL` في `.env` أو `.env.local`
- عرض رسائل الخطأ بشكل أوضح للمستخدم (مثلاً "قاعدة البيانات غير متاحة" بدلاً من رسالة عامة)
- التحقق من أن الباك اند يعمل قبل إظهار صفحة التسجيل أو إضافة فحص صحة الـ API

  3. ربط `handleVoiceMessage` في `ChatWindow` بعملية الإرسال الفعلية

## 2. نقل عمود المحادثات إلى اليمين (للمحتوى العربي)

**الوضع الحالي:** التطبيق يستخدم `DirectionProvider` مع `defaultDirection="rtl"`، و`App.css` يحتوي على:

```css
[dir="rtl"] .app {
  flex-direction: row-reverse;
}
```

مع `row-reverse`، الـ sidebar يظهر على اليمين. إذا كان يظهر على اليسار، قد يكون السبب:
- تخزين `wareed_direction` في `localStorage` كـ `ltr`
- أو عدم تطبيق `dir="rtl"` على العنصر الصحيح

**الحل:**
1. التأكد من أن `DirectionProvider` يغلف التطبيق وأن `defaultDirection="rtl"` مضبوط.
2. إضافة قاعدة صريحة لوضع الـ sidebar على اليمين دائماً في الوضع العربي:

```css
/* ChatSidebar.css - وضع الـ sidebar على اليمين للمحتوى العربي */
.chat-sidebar {
  order: 1; /* أو استخدام margin-inline-start: auto حسب التخطيط */
}

/* أو في App.css - فرض الـ sidebar على اليمين */
.app[dir="rtl"] .chat-sidebar {
  order: 2; /* يظهر بعد نافذة المحادثة في الترتيب البصري = على اليمين */
}
.app[dir="rtl"] .chat-window {
  order: 1;
}
```

أو الاعتماد على `flex-direction: row-reverse` مع التأكد من أن `dir="rtl"` مضبوط على `document.documentElement` أو `#root`.

| العنصر | الملاحظة | الاقتراح |

### أولوية منخفضة

6. **إدراج الصور والمرفقات** — يحتاج قراراً وتعديلات في الباك اند.

8. **اختيار اللغة** — دعم متعدد اللغات.

## 6. ملاحظات إضافية

- **Proxy:** في `package.json` الـ proxy مضبوط على `http://127.0.0.1:8000`. تأكد أن الباك اند يعمل على نفس العنوان والمنفذ.
- **CORS:** الباك اند يسمح بـ `localhost:3000`. إذا استخدمت منفذاً أو نطاقاً آخر، قد تحتاج تحديث `CORS_ORIGINS`.
- **التوثيق:** الـ API موثّق في `/docs` (Swagger) عند تشغيل الباك اند.
